Leer más

Durante el último período, la actividad de amenazas se ha centrado en vectores de acceso inicial ya conocidos pero con mayor sofisticación. La vulnerabilidad de inyección SQL en Fortinet FortiClient EMS representa un riesgo directo para la infraestructura expuesta a internet, permitiendo la ejecución de código no autorizado. En el ámbito del correo electrónico, la integración de Inteligencia Artificial está generando campañas de phishing más convincentes y a escala, apuntando a credenciales de Office 365 y explotando flujos de inicio de sesión de confianza. El ransomware mantiene su volumen, con un cambio táctico hacia la doble extorsión (robo de datos más cifrado) como estándar. Se identificó una campaña de compromiso de la cadena de suministro de npm (axios) que distribuye RATs para Windows, macOS y Linux, lo que subraya la necesidad de una vigilancia constante sobre las dependencias de software. Aunque no se reportaron nuevos CVEs confirmados para otros productos específicos del stack, la ejecución de sistemas operativos y bases de datos obsoletas como Windows XP/7 y SQL Server Express 2005/2008 sigue siendo un riesgo latente de explotación de vulnerabilidades conocidas.

Incidentes Reportados

Compromiso de Cadena de Suministro de npm (Axios)

Internet-exposed RCE Explotado activamente Directa

Publicado: 2026-03-31

  • El paquete npm `axios` fue comprometido a través de un secuestro de cuenta de mantenedor, inyectando versiones maliciosas.
  • Se desplegaron RATs (Remote Access Trojans) específicos para la plataforma: PowerShell para Windows, Mach-O C++ para macOS y Python para Linux.
  • El RAT de macOS ha sido clasificado como NukeSped, vinculado a las actividades del grupo BlueNoroff (Lazarus-exclusive).
  • Microsoft rastrea la amenaza para macOS como Backdoor:Mac/Axios.A, creando una familia de detección dedicada.

Acción Recomendada: Revisar y auditar dependencias de paquetes npm. Implementar políticas de integridad de software. Asegurar que las soluciones EDR/XDR estén actualizadas con las últimas firmas para detectar RATs en sistemas Windows, macOS y Linux. Considerar el uso de herramientas de escaneo de la cadena de suministro.

Nuevas Vulnerabilidades (CVEs)

CVE-2026-21643: Inyección SQL en Fortinet FortiClient EMS

Internet-exposed RCE Explotado activamente Indirecta

Publicado: 2026-04-13 (CISA KEV)

  • Vulnerabilidad de inyección SQL en Fortinet FortiClient EMS que podría permitir a un atacante no autenticado ejecutar código o comandos no autorizados.
  • Requiere peticiones HTTP específicamente diseñadas para su explotación.
  • CISA ha añadido esta vulnerabilidad a su catálogo de vulnerabilidades explotadas conocidas (KEV) el 13 de abril de 2026, citando evidencia de explotación activa.

Acción Recomendada: Actualizar Fortinet FortiClient EMS a la versión parcheada más reciente tan pronto como sea posible. Revisar los registros para detectar intentos de explotación. Implementar segmentación de red para limitar el acceso a FortiClient EMS desde internet si es posible.

CVE-2012-1854: Vulnerabilidad de carga de librerías inseguras en Microsoft Visual Basic for Applications

Internal RCE Explotado activamente Directa

Actividad de Ransomware

Panorama del Ransomware: Nuevo Normal y Doble Extorsión

Internet-exposed Info Leak Explotado activamente Indirecta

Publicado: 2026-04-16

  • La actividad de ransomware se ha estabilizado en un "nuevo normal" durante el primer trimestre de 2026, con volúmenes consistentes respecto a trimestres anteriores.
  • Grupos como The Gentlemen, Qilin, Akira, Clop, INC Ransom, Play, DragonForce y Sinobi siguen siendo prominentes, con un aumento en la eficiencia y profesionalización.
  • El enfoque en la exfiltración de datos antes del cifrado (doble extorsión) se ha convertido en una táctica estándar en el 74% de los ataques, lo que hace que las copias de seguridad por sí solas sean insuficientes como defensa.
  • La vulnerabilidad en Microsoft Exchange Server (CVE-2023-21529) ha sido utilizada en ataques para desplegar ransomware Medusa.

Acción Recomendada: Fortalecer las defensas contra el acceso inicial (phishing, vulnerabilidades internet-exposed). Implementar monitoreo de red para detectar exfiltración de datos. Mantener planes de recuperación ante desastres actualizados que consideren la pérdida de datos y no solo el cifrado. Asegurar la aplicación de parches en todos los sistemas operativos y aplicaciones, especialmente en servidores Exchange.

Recomendaciones Operativas Clave

Gestión de Parches y Actualizaciones Críticas

  • Fortinet FortiClient EMS: Aplicar inmediatamente las actualizaciones disponibles para mitigar la vulnerabilidad CVE-2026-21643.
  • Sistemas Operativos Windows: Asegurar que Windows 10, 11, Server 2012 y 2016 tienen aplicadas todas las actualizaciones de seguridad más recientes de Microsoft para abordar las vulnerabilidades explotadas activamente.
  • Microsoft Office / VBA: Mantener las aplicaciones de Office actualizadas y educar a los usuarios sobre los riesgos de habilitar macros o abrir archivos de fuentes no confiables, dada la explotación activa de CVE-2012-1854.
  • Proxmox VE, Ubuntu Server, Red Hat Enterprise Linux: Implementar un ciclo regular de aplicación de parches y actualizaciones para todas las instancias, verificando periódicamente las notas de lanzamiento de seguridad de los proveedores (pve.proxmox.com, ubuntu.com/security, access.redhat.com/security).
  • Dell iDRAC, Optiplex, Latitude: Monitorear regularmente support.dell.com para nuevas versiones de firmware y aplicarlas siguiendo los procedimientos del fabricante.
  • Zimbra Collaboration Professional, Zabbix: Mantener estas plataformas actualizadas a las últimas versiones estables y aplicar parches de seguridad. Revisar los avisos de seguridad de zimbra.com y zabbix.com.

Defensa contra Phishing y Cadenas de Suministro

  • Concienciación y Formación Anti-Phishing: Intensificar la formación de usuarios para identificar correos de phishing, especialmente aquellos generados por IA, y fomentar la notificación de correos sospechosos.
  • Protección de Office 365: Implementar autenticación multifactor (MFA) resistente a phishing (e.g., FIDO2), monitoreo de actividad de inicio de sesión anómala y uso de Microsoft Defender for Office 365 para detección avanzada de amenazas.
  • Seguridad en la Cadena de Suministro: Auditar regularmente las dependencias de software, especialmente paquetes de npm y otros repositorios públicos, para detectar inclusiones maliciosas como la observada en el paquete `axios`.
  • Políticas de Acceso: Reforzar los principios de mínimo privilegio y "Zero Trust" para reducir el impacto de una posible explotación de credenciales o sistemas comprometidos.

Mitigación de Ransomware y Resiliencia

  • Copias de Seguridad Inmutables: Asegurar que las copias de seguridad críticas se realicen de forma inmutable o fuera de línea para protegerse contra la exfiltración y el cifrado.
  • Segmentación de Red: Implementar una segmentación de red granular para limitar la propagación lateral de ransomware y aislar sistemas críticos.
  • Detección y Respuesta (EDR/XDR): Mantener soluciones ESET Antivirus corporativo (u otras soluciones EDR/XDR) actualizadas y configuradas para detectar comportamientos anómalos y actividad de ransomware.
  • Monitoreo de la Superficie de Ataque: Escanear y monitorear regularmente la superficie de ataque expuesta a internet para identificar y remediar vulnerabilidades antes de que sean explotadas.

Gestión de Sistemas Obsoletos y IoT

  • SQL Server Express 2005 y 2008: Si aún están en uso, planificar la migración a versiones más recientes y soportadas. Implementar controles de acceso estrictos y segmentación de red para aislar estas instancias.
  • Windows XP y 7: Priorizar la migración de estos sistemas a versiones modernas y soportadas. Si la migración no es posible, aislarlos de la red principal y aplicar controles de seguridad compensatorios estrictos.
  • Cámaras y HVR Dahua: Cambiar las contraseñas predeterminadas o débiles. Asegurar que el firmware esté actualizado. Segmentar estos dispositivos en una red IoT dedicada y restringir su acceso a internet solo a lo estrictamente necesario.
