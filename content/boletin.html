Radar Operativo Diario

  • Internet-exposed: Incremento en el escaneo de servicios RDP y SSH en entornos Windows Server y RHEL.
  • Correo / Phishing: Campañas activas de spear-phishing dirigidas a credenciales de Office 365.
  • IoT / Cámaras: Observación de intentos de acceso no autorizado a dispositivos Dahua expuestos con credenciales débiles.
  • Ransomware: Aumento en la actividad de grupos que explotan vulnerabilidades de VPNs y RDP para acceso inicial.
Leer más

Durante el último período, se ha observado un incremento en el volumen de escaneo automatizado dirigido a puertos RDP (3389) y SSH (22) en redes con infraestructura Windows Server (incluyendo 2008 y 2012) y RHEL (7 y 8) expuestas a internet. Esto indica una preparación para ataques de fuerza bruta o explotación de vulnerabilidades conocidas. Las campañas de phishing contra Office 365 persisten con nuevas técnicas de evasión, enfocándose en la captura de credenciales y tokens de sesión. En el segmento IoT, dispositivos Dahua mal configurados o con firmware desactualizado siguen siendo objetivos fáciles para accesos no autorizados. La amenaza de ransomware continúa evolucionando, con grupos focalizándose en accesos iniciales a través de VPNs (incluyendo FortiGate si no está debidamente actualizado) y el abuso de RDP.

Vulnerabilidades Críticas (CVEs)

FortiOS - Vulnerabilidad de inyección de comandos OS (CVE-2024-21762)

Internet-exposed RCE Explotado activamente Directa

Publicado: 2024-02-09 (Relevante para stack por explotación activa)

  • Afecta a FortiOS 7.2.12 y versiones anteriores, incluyendo FortiProxy y FortiClientEMS.
  • Permite la ejecución remota de comandos (RCE) no autenticada.
  • Fortinet ha confirmado la explotación activa de esta vulnerabilidad.
  • La vulnerabilidad reside en la interfaz SSL VPN, si está expuesta a internet.

Acción Recomendada: Actualizar FortiOS a la versión 7.4.2, 7.2.8, 7.0.14, 6.4.15 o 6.2.16 (o superior según el ciclo de vida). Si la actualización no es posible, deshabilitar la interfaz SSL VPN o restringir el acceso a direcciones IP de confianza. Se recomienda un escaneo inmediato de todos los dispositivos FortiGate expuestos.

Zimbra Collaboration - Inyección de comandos en ZCS (CVE-2023-34390)

Internet-exposed RCE Observado Directa

Publicado: 2023-06-20 (Explotación activa observada en 2024)

  • Vulnerabilidad de inyección de comandos OS en Zimbra Collaboration 10 y versiones anteriores.
  • Puede ser explotada por un atacante no autenticado para ejecutar comandos arbitrarios.
  • La explotación activa ha sido observada, lo que subraya la necesidad de parchar incluso versiones más antiguas.

Acción Recomendada: Asegurarse de que Zimbra Collaboration Professional 10 y versiones anteriores estén actualizadas con los parches más recientes. Realizar una auditoría de logs para detectar actividades anómalas en el servidor Zimbra.

Actividad Ransomware

Tendencias de Ransomware dirigidas a Servicios de Acceso Remoto

Internet-exposed Ransomware Observado Indirecta

Publicado: 2026-03-27

  • Grupos de ransomware como LockBit y BlackCat continúan explotando vulnerabilidades en servicios VPN (incluyendo FortiGate y OpenVPN en RHEL/Ubuntu) y credenciales débiles/robadas de RDP en sistemas Windows Server.
  • El vector de ataque principal sigue siendo el acceso inicial a través de estas puertas de enlace expuestas.
  • Se observa una mayor sofisticación en la fase de movimiento lateral y exfiltración de datos antes del cifrado.

Acción Recomendada: Implementar MFA en todos los servicios de acceso remoto (VPN, RDP, Office 365, Proxmox VE). Reforzar políticas de contraseñas complejas. Monitorear logs de autenticación anómalos. Asegurarse de que las copias de seguridad estén aisladas y probadas periódicamente. Parchear inmediatamente cualquier vulnerabilidad conocida en dispositivos de borde.

Recomendaciones Generales y Mitigaciones

Actualización y Hardening de Dispositivos de Borde

Aplicable a: FortiGate, Proxmox VE, Zimbra

  • FortiGate: Priorizar la actualización a las últimas versiones de FortiOS (7.4.2, 7.2.8, etc.) para mitigar CVEs críticos. Deshabilitar la interfaz SSL VPN si no es esencial o limitar el acceso vía IP.
  • Proxmox VE 8.4.16+: Mantener el hypervisor y sus componentes (QEMU, KVM) actualizados. Implementar autenticación de dos factores (2FA) para el acceso a la interfaz de gestión.
  • Zimbra Collaboration Professional 10: Aplicar todos los parches de seguridad liberados por el vendor. Considerar la segmentación de red del servidor Zimbra.

Acción Recomendada: Revisar el plan de parches, priorizando dispositivos perimetrales. Implementar MFA donde sea posible. Consultar guías de hardening específicas del vendor.

Gestión de Parches para Sistemas Operativos y Aplicaciones

Aplicable a: Windows XP, 7, 10, 11, Server 2003, 2008, 2012, 2016, RHEL 7 y 8, Ubuntu Server 24.04 LTS, SQL Server Express, Office 365

  • Microsoft Systems: Establecer un ciclo de aplicación de parches mensual para todas las estaciones de trabajo y servidores Windows. Prestar atención a los boletines de seguridad de SQL Server Express.
  • Linux Servers: Mantener los sistemas RHEL 7/8 y Ubuntu Server 24.04 LTS actualizados con los últimos parches de seguridad a través de yum/dnf o apt.
  • Office 365: Utilizar las capacidades de seguridad nativas (MFA, ATP, DLP). Concienciación continua sobre phishing para usuarios.

Acción Recomendada: Asegurar la automatización y verificación de la aplicación de parches en todos los sistemas operativos y software. Reforzar la formación de usuarios en seguridad.

Guía de Actualizaciones de Seguridad de Microsoft Red Hat Security Updates Ubuntu Security Notices

Seguridad para Endpoints y IoT

Aplicable a: ESET Antivirus, Dell Optiplex/Latitude, macOS, Cámaras Dahua

  • ESET Antivirus: Asegurar que la suite corporativa de ESET esté actualizada y las políticas de detección y respuesta sean óptimas en todos los endpoints (Windows, macOS).
  • Dell Firmware: Actualizar el firmware de Dell Optiplex y Latitude, incluyendo Dell iDRAC (hasta Gen 13), a las últimas versiones estables para corregir vulnerabilidades de bajo nivel.
  • Cámaras Dahua: Cambiar credenciales por defecto, actualizar firmware, segmentar las redes de cámaras y restringir el acceso a internet.

Acción Recomendada: Implementar un plan de gestión de vulnerabilidades para firmware y software de seguridad. Auditar la configuración de dispositivos IoT.
