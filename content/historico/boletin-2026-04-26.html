Radar Operativo Diario

  • Internet-exposed: Riesgo continuo en sistemas operativos y bases de datos heredados sin soporte oficial (Windows XP, Server 2003/2008, SQL Server Express 2005/2008) y dispositivos perimetrales con configuraciones subóptimas.
  • Correo / Phishing: Campañas persistentes de phishing dirigidas a credenciales de Office 365, incluyendo ataques sofisticados que evaden controles tradicionales y explotan la autenticación de código de dispositivo.
  • IoT / Cámaras: Exposición de cámaras y HVR Dahua a accesos no autorizados debido a credenciales predeterminadas o débiles, así como vulnerabilidades de firmware conocidas y explotadas.
  • Ransomware: Actividad global moderada, con grupos que siguen priorizando la explotación de vulnerabilidades conocidas en sistemas sin parches y el uso de la doble o triple extorsión.
Leer más

El período reciente ha sido relativamente estable, sin la aparición de vulnerabilidades críticas nuevas y ampliamente explotadas que afecten directamente al stack tecnológico definido en el último mes. Sin embargo, la persistencia de entornos con sistemas operativos y aplicaciones heredadas, como Windows XP, Windows Server 2003/2008, y SQL Server Express 2005/2008, representa un vector de ataque constante. Estos sistemas, al estar al final de su vida útil (EOL), no reciben actualizaciones de seguridad, lo que los convierte en objetivos fáciles para atacantes que explotan vulnerabilidades ya conocidas o debilidades de configuración. Las campañas de phishing contra Office 365 siguen siendo una amenaza predominante, con técnicas cada vez más sofisticadas, incluyendo el abuso de la infraestructura de Microsoft y el phishing de código de dispositivo para eludir la detección y la autenticación multifactor (MFA). La seguridad de los dispositivos IoT, como las cámaras Dahua, requiere atención continua, ya que las configuraciones predeterminadas o credenciales débiles y las vulnerabilidades de firmware conocidas siguen siendo puntos de entrada para accesos no autorizados y control remoto. La actividad de ransomware, aunque moderada, continúa enfocándose en la explotación de vulnerabilidades conocidas y la implementación de tácticas de doble extorsión, donde los datos son exfiltrados y luego encriptados, aumentando la presión sobre las víctimas para el pago. Se recomienda mantener una vigilancia activa sobre los sistemas de monitoreo y asegurar que todas las plataformas críticas estén protegidas con las últimas actualizaciones de seguridad y configuraciones robustas.

Recomendaciones Operativas

Gestión de Sistemas End-of-Life (EOL)

Internal RCE Explotado activamente Directa

Publicado: 2026-04-26

  • Realice un inventario exhaustivo de todos los sistemas con software EOL (Windows XP, Server 2003/2008, SQL Server Express 2005/2008).
  • Establezca un plan de migración o reemplazo prioritario para estos sistemas.
  • Si la migración no es inmediata, aisle los sistemas EOL en segmentos de red dedicados (VLANs) con controles de acceso estrictos para limitar la exposición.
  • Implemente monitoreo avanzado para detectar cualquier actividad anómala en estos segmentos.
CISA: Transición de Software EOL Microsoft: Fin de Soporte para Windows Server

Refuerzo contra Phishing en Office 365

Internet-exposed Phishing Explotado activamente Directa

Publicado: 2026-04-26

  • Eduque y capacite continuamente a los usuarios sobre las últimas tácticas de phishing, incluyendo suplantación de identidad de dominios internos y phishing de código de dispositivo.
  • Asegure que la autenticación multifactor (MFA) esté habilitada y aplicada en todas las cuentas de Office 365, con monitoreo de intentos de bypass de MFA.
  • Configure y revise políticas anti-phishing en Microsoft Defender para Office 365, prestando atención a la inteligencia de suplantación y los indicadores de remitentes no autenticados.
  • Considere la implementación de soluciones de seguridad de correo electrónico adicionales que complementen las capacidades nativas de Office 365.

Seguridad de Cámaras y HVR Dahua

Internet-exposed Auth Bypass Explotado activamente Directa

Publicado: 2026-04-26

  • Cambie todas las contraseñas predeterminadas y utilice credenciales fuertes y únicas para todos los dispositivos Dahua.
  • Actualice el firmware de todas las cámaras y HVR Dahua a la última versión disponible del fabricante para parchear vulnerabilidades conocidas, incluyendo CVE-2021-33045 y otras que permiten el bypass de autenticación o acceso no autorizado.
  • Evite exponer la interfaz web de los dispositivos Dahua directamente a internet. Utilice VPN para acceso remoto seguro o restrinja el acceso a IPs de confianza.
  • Deshabilite UPnP y elimine cualquier regla de reenvío de puertos innecesaria.
