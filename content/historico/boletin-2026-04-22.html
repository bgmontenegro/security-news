Actualizaciones de Seguridad Críticas de Microsoft (Abril 2026)

Internal Internet-exposed RCE Privilege Escalation Explotado activamente Directa

Publicado: 2026-04-14

  • Microsoft ha lanzado el Patch Tuesday de abril de 2026, corrigiendo 167 vulnerabilidades de seguridad en Windows y software relacionado.
  • Entre las correcciones se incluye un zero-day en SharePoint Server y una debilidad públicamente divulgada en Windows Defender, conocida como "BlueHammer".
  • La amplia naturaleza de estas vulnerabilidades afecta a diversas versiones de Windows (10, 11) y Windows Server (2012, 2016), así como a productos de Office 365.
  • La explotación de zero-days, como el de SharePoint, indica un riesgo inmediato para los sistemas expuestos o que procesan contenido externo.

Acción Recomendada: Priorizar la aplicación de todas las actualizaciones de seguridad de Microsoft de abril de 2026 en los sistemas operativos Windows y servidores, incluyendo SharePoint y Office 365. Realizar pruebas de regresión en entornos de desarrollo antes de la implementación en producción.

Campaña de Phishing dirigida a credenciales de Office 365

Internet-exposed Phishing Observado Directa

Publicado: 2026-03-30

  • Se ha identificado una campaña de phishing que utiliza la suplantación de identidad para engañar a usuarios de Microsoft 365 y robar sus credenciales.
  • La táctica incluye la creación de páginas falsas que imitan los portales de inicio de sesión de Office 365, con mensajes de urgencia o alertas de cuenta.
  • Esta campaña destaca la sofisticación creciente de los ataques de ingeniería social para evadir las defensas de correo electrónico.

Acción Recomendada: Reforzar la formación de concienciación sobre phishing para todos los usuarios de Office 365, haciendo hincapié en la verificación de URLs y la alerta ante mensajes sospechosos. Implementar y hacer cumplir la autenticación multifactor (MFA) para todas las cuentas de Office 365. Revisar las configuraciones de seguridad de ESET Antivirus y Office 365 para la detección de URL maliciosas y suplantación de identidad.

Nuevos y Relevantes CVEs

Persistencia de la Vulnerabilidad MS17-010 (EternalBlue)

Internal Internet-exposed RCE Explotado activamente Directa

Publicado: 2026-03-28

  • Aunque es una vulnerabilidad antigua, MS17-010 (conocida como EternalBlue) continúa siendo relevante en análisis de seguridad recientes, lo que indica su persistencia como vector de ataque.
  • Permite la ejecución remota de código sin autenticación en sistemas Windows que utilizan SMBv1 y no están parcheados, como Windows 7 y Windows Server 2008.
  • La explotación de EternalBlue fue la base de ataques de ransomware a gran escala como WannaCry, y sigue siendo una amenaza para infraestructuras desactualizadas.

Acción Recomendada: Verificar y asegurar que todos los sistemas Windows 7, Windows Server 2008 y cualquier otro sistema susceptible (incluyendo Windows XP/Server 2003, aunque no directamente mencionados en la fuente, son vulnerables a SMBv1) estén completamente parcheados contra MS17-010. Deshabilitar el protocolo SMBv1 en todos los sistemas donde no sea estrictamente necesario. Segmentar redes para aislar sistemas legacy y limitar su exposición.

Riesgos en macOS por Scams de Aplicaciones Falsas

User Interaction Malware Observado Directa

Publicado: 2026-04-02

  • Se ha observado un aumento en las estafas que utilizan aplicaciones de IA y herramientas de limpieza falsas para engañar a los usuarios de macOS y que instalen malware.
  • Estas campañas se basan en la ingeniería social para persuadir a los usuarios a ejecutar software malicioso por sí mismos.

Acción Recomendada: Educar a los usuarios de macOS sobre los riesgos de descargar software de fuentes no verificadas y sobre cómo identificar aplicaciones falsas. Fomentar la descarga de aplicaciones solo desde la App Store oficial o desarrolladores de confianza. Asegurar que ESET Antivirus esté actualizado y configurado para escanear descargas.

Recomendaciones Operativas Clave

Gestión de Parches y Actualizaciones

  • Windows y Office 365: Aplicar de inmediato todas las actualizaciones de seguridad de Microsoft publicadas en el Patch Tuesday de abril de 2026 para Windows 10, 11, Windows Server 2012, 2016 y Office 365. Prestar especial atención a las correcciones para SharePoint Server y Windows Defender.
  • Sistemas Windows Legacy: Verificar la aplicación de parches para MS17-010 (EternalBlue) en Windows 7 y Windows Server 2008. Deshabilitar SMBv1 si no es esencial. Considerar la migración de Windows XP y Server 2003, o aplicar una segmentación de red estricta.
  • macOS: Mantener los sistemas macOS actualizados con las últimas versiones de seguridad de Apple. Educar a los usuarios sobre los riesgos de software no oficial y estafas.
  • Linux (Ubuntu Server 24.04 LTS, RHEL 7/8): Realizar periódicamente las actualizaciones de seguridad del sistema operativo y las aplicaciones.

Seguridad del Correo Electrónico y Concienciación

  • Office 365: Reforzar la autenticación multifactor (MFA) para todas las cuentas de Office 365. Revisar las políticas anti-phishing y anti-spam en el centro de administración de seguridad de Office 365.
  • Formación: Realizar campañas de concienciación sobre phishing, especialmente dirigidas a la identificación de correos de suplantación de identidad que buscan credenciales.
  • ESET Antivirus: Asegurar que las definiciones de ESET Antivirus corporativo estén siempre actualizadas y que las configuraciones de protección de correo electrónico y web estén optimizadas.

Hardening de Infraestructura y Segmentación

  • iDRAC y FortiGate: Revisar las configuraciones de todos los dispositivos FortiGate y Dell iDRAC. Asegurar que los puertos de administración no estén expuestos directamente a Internet y que se utilicen credenciales robustas y MFA. Actualizar el firmware de iDRAC a la última versión disponible para la generación 13.
  • Zimbra Collaboration Professional: Mantener Zimbra Collaboration Professional 10 actualizado con los últimos parches de seguridad para mitigar vulnerabilidades conocidas.
  • Proxmox VE y Zabbix: Asegurar que las instalaciones de Proxmox VE 8.4.16 y Zabbix 7 estén actualizadas y que sus interfaces de administración estén protegidas con acceso restringido y credenciales fuertes.
  • Cámaras Dahua: Cambiar todas las contraseñas predeterminadas y deshabilitar servicios innecesarios en cámaras y HVR Dahua. Mantener el firmware actualizado si hay parches de seguridad disponibles.
