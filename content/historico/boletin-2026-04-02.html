Durante el período reciente, el panorama de amenazas muestra una continuidad en la explotación de vectores de ataque conocidos y el enfoque en sistemas legados. Se observa una actividad constante de escaneo y ataque a dispositivos expuestos a Internet, especialmente cámaras Dahua y módulos iDRAC de Dell, aprovechando credenciales débiles o vulnerabilidades ya publicadas. Las campañas de phishing dirigidas a usuarios de Office 365 se mantienen en niveles elevados, con una sofisticación creciente en la suplantación de identidad de Microsoft para el robo de credenciales y la toma de control de cuentas. La principal preocupación operativa recae en la alta exposición de los sistemas con fin de vida útil (EOL), como Windows Server 2003, 2008 y 2012, así como SQL Server Express 2005 y 2008. Estos sistemas carecen de soporte y actualizaciones de seguridad, convirtiéndolos en blancos fáciles y persistentes para ataques de ransomware y otras amenazas avanzadas. La mitigación activa a través de la actualización de firmware, la implementación de MFA, la segmentación de red y la migración de sistemas legados son acciones críticas para reducir la superficie de ataque.

Internet-exposed RCE Observado Directa

Actividad de Ransomware

Sistemas EOL de Windows Server y SQL Server como objetivo principal

Internal Ransomware Explotado activamente Directa

Publicado: 2023-10-10 (EOL Windows Server 2012)

  • Los sistemas Windows Server 2003, 2008 y 2012/2012 R2, así como SQL Server Express 2005 y 2008, al haber alcanzado su fin de vida útil (EOL), carecen de actualizaciones de seguridad, lo que los convierte en objetivos preferentes para ataques de ransomware.
  • La ejecución de software sin soporte implica riesgos significativos de seguridad y cumplimiento, y puede anular la cobertura del seguro cibernético.
  • Grupos de ransomware explotan activamente las vulnerabilidades conocidas y no parcheadas en estos entornos legados para establecer persistencia y propagar ataques.

Acción Recomendada: Priorizar la migración de todos los sistemas operativos y bases de datos con EOL a versiones soportadas. Si la migración no es posible de inmediato, implementar las Actualizaciones de Seguridad Extendidas (ESU) donde estén disponibles y reforzar la segmentación de red, así como las soluciones de respaldo inmutables.

Recomendaciones y Mejores Prácticas

Actualización y Reforzamiento de FortiOS

Internal Configuración Teórico Directa

Publicado: 2025-02-13 (última actualización sobre FortiGate)

  • Asegurarse de que todos los dispositivos FortiGate, FortiAP y FortiMail ejecuten FortiOS 7.2.12 o posterior.
  • Implementar perfiles de seguridad robustos, incluyendo IPS/IDS, Antivirus y Filtrado Web en todas las políticas de tráfico relevante.
  • Revisar regularmente las configuraciones del firewall para evitar políticas amplias o errores que puedan generar problemas de conectividad o abrir la red.
  • Considerar la habilitación de la inspección SSL/TLS para AV e IPS para una visibilidad completa del tráfico cifrado.

Acción Recomendada: Programar revisiones periódicas de políticas y configuraciones de Fortinet. Mantener los sistemas actualizados y consultar FortiGuard Labs para las últimas amenazas y parches.

Reforzamiento de Proxmox VE 8.4.16

Internal Configuración Teórico Directa

Publicado: 2026-03-30 (Guía de Seguridad y Mejores Prácticas)

  • Mantener Proxmox VE 8.4.16 y sus sistemas operativos invitados actualizados con los últimos parches de seguridad.
  • Utilizar contraseñas complejas para el usuario root y otras credenciales, y configurar la autenticación de dos factores (2FA).
  • Implementar un firewall robusto (basado en iptables) para limitar el acceso a puertos esenciales (SSH, interfaz web Proxmox, VNC).
  • Configurar políticas de backup automáticas y proteger los backups contra ransomware.
  • Asegurar que los servicios innecesarios estén deshabilitados para reducir la superficie de ataque.

Acción Recomendada: Realizar auditorías de seguridad periódicas del entorno Proxmox. Consultar la documentación oficial de Proxmox para las mejores prácticas de seguridad y endurecimiento.

Endurecimiento de Ubuntu Server 24.04 LTS y RHEL 7/8

Internal Configuración Teórico Directa

Publicado: 2025-03-20 (Ubuntu hardening)

  • Implementar autenticación basada en claves SSH y deshabilitar el inicio de sesión de root.
  • Configurar firewalls (UFW para Ubuntu, FirewallD para RHEL) para permitir solo el tráfico necesario.
  • Utilizar herramientas como Fail2Ban para la prevención de intrusiones y la protección contra ataques de fuerza bruta.
  • Mantener el sistema operativo y todas las aplicaciones actualizadas con parches de seguridad.
  • Considerar la aplicación de los benchmarks de seguridad del CIS para Ubuntu 24.04 LTS y RHEL 7/8.

Acción Recomendada: Seguir guías de endurecimiento específicas para cada distribución y versión. Establecer rutinas de auditoría y monitoreo de la seguridad de los servidores Linux.
