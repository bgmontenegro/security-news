Windows Server 2008 Windows Server 2012 SMB RDP

Resurgimiento de Ataques SMB/RDP contra Servidores Windows Antiguos

Publicado: 2026-03-08

  • Grupos de ransomware están intensificando los ataques contra servidores Windows 2008 y 2012 que carecen de los últimos parches de seguridad para SMB y RDP.
  • Exploits como BlueKeep (CVE-2019-0708) y EternalBlue (CVE-2017-0144) siguen siendo vectores de entrada eficaces en entornos no parcheados.
  • Los compromisos suelen llevar al cifrado completo de la red.

Acción Recomendada: Aplicar parches de seguridad extendidos si están disponibles, migrar a versiones más recientes de Windows Server, y endurecer la seguridad de RDP y SMB (deshabilitar SMBv1, restringir acceso a RDP).

Parcheo Actualización

Priorizar la Actualización y Parcheo Urgente

Seguridad de Acceso Red

Endurecimiento de Acceso y Segmentación de Red

Publicado: 2026-03-26

  • Dell iDRAC, FortiGate: Restringir el acceso a las interfaces de administración solo a IPs de confianza, preferiblemente a través de VPN.
  • Cámaras Dahua: Aislar cámaras en una VLAN separada. Deshabilitar UPnP y cualquier puerto innecesario.
  • Broadcast Electronics: Asegurar que los equipos de automatización no estén directamente expuestos a internet.
  • RDP/SMB: Deshabilitar SMBv1. Restringir el acceso a RDP desde internet y aplicar autenticación multifactor.
  • SQL Server Express: Asegurar que las instancias de bases de datos no sean accesibles directamente desde internet y usar contraseñas robustas.

Acción Recomendada: Revisar la arquitectura de red y las políticas de firewall. Implementar segmentación de red para separar sistemas críticos y dispositivos IoT.

Mitigación Prevención

Concienciación y Respaldo de Datos

Publicado: 2026-03-26

  • ESET Antivirus: Verificar que las definiciones estén actualizadas en todos los dispositivos Windows y macOS. Realizar escaneos completos periódicamente.
