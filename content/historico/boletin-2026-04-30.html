El período actual, hasta el 30 de abril de 2026, ha sido relativamente estable en cuanto a la aparición de nuevas vulnerabilidades críticas (Zero-Days) que impacten directamente y de forma masiva las versiones más recientes y actualizadas de nuestro stack tecnológico (FortiOS 7.2.12+, Proxmox VE 8.4.16+, Zabbix 7+, Ubuntu Server 24.04 LTS, Zimbra Collaboration Professional 10+).

Sin embargo, la principal preocupación operativa sigue siendo la exposición y explotación de sistemas End-of-Life (EOL) como Windows XP, 7, Windows Server 2003, 2008, 2012 y SQL Server Express 2005/2008. Estos sistemas, al no recibir actualizaciones de seguridad, son un objetivo constante y de bajo costo para los atacantes, siendo vulnerables a exploits conocidos y amenazas de ransomware.

Las campañas de phishing dirigidas a entornos de Office 365 continúan evolucionando, con tácticas más sofisticadas que logran evadir las defensas tradicionales, incluyendo la suplantación de identidad interna y el uso de plataformas PhaaS. Asimismo, los dispositivos IoT, como las cámaras Dahua con firmware desactualizado o credenciales predeterminadas, representan un riesgo significativo de ser incorporados a botnets o ser explotados para accesos no autorizados.

La consolidación de la seguridad y el enfoque en la resiliencia son tendencias clave observadas para 2026, lo que refuerza la necesidad de mantener todo el stack tecnológico actualizado, segmentado y protegido con autenticación robusta y monitoreo constante.

Vulnerabilidades (CVEs)

CVE-2026-32202: Filtración de Hash NTLM en Windows

Internet-exposed Info Leak Explotado activamente Directa

Publicado: 2026-04-30

  • Una vulnerabilidad de "zero-click" en Windows, identificada como CVE-2026-32202, permite la filtración de hashes NTLM.
  • Esta vulnerabilidad fue reportada por Akamai como un parche incompleto de una RCE anterior (CVE-2026-21510) y ha sido explotada en ataques de Zero-Day.
  • Los atacantes pueden usar los hashes NTLM robados en ataques "pass-the-hash" para autenticarse como el usuario comprometido, permitiendo movimiento lateral y robo de datos.
  • CISA ha añadido CVE-2026-32202 a su Catálogo de Vulnerabilidades Explotadas Conocidas (KEV), ordenando a las agencias federales parchear sus sistemas Windows.

Acción Recomendada: Aplicar inmediatamente el parche de seguridad de Microsoft para CVE-2026-32202. Priorizar la actualización de todos los sistemas Windows (10, 11, Server 2012, 2016) expuestos a la red o con interacción de usuarios.

Explotación de vulnerabilidades conocidas en cámaras Dahua

Internet-exposed Auth Bypass Observado Directa

Publicado: 2025-07-31

  • Se sigue observando la explotación de vulnerabilidades conocidas (ej. CVE-2025-31700, CVE-2025-31701) en modelos de cámaras Dahua con firmware anterior a abril de 2025.
  • Estas fallas permiten el acceso no autorizado a interfaces web/RTSP, ejecución de comandos con privilegios elevados y elusión de mecanismos de autenticación.
  • Dispositivos sin parches y con credenciales débiles son blanco de botnets y accesos ilegítimos.

Acción Recomendada: Actualizar inmediatamente el firmware de todas las cámaras Dahua a la última versión disponible en el sitio oficial del fabricante. Implementar contraseñas fuertes y únicas, y aislar los dispositivos IoT en segmentos de red dedicados.

Actividad de Ransomware

Ransomware dirigido a sistemas Windows Server y SQL Server EOL

Internet-exposed Privilege Escalation Explotado activamente Directa

Publicado: 2026-04-30

  • Los sistemas operativos Windows Server (2003, 2008, 2012) y SQL Server Express (2005, 2008) en fin de vida útil (EOL) son un objetivo primario y constante para los ataques de ransomware.
  • La falta de parches de seguridad para estos sistemas permite a los atacantes explotar vulnerabilidades conocidas, facilitando el acceso inicial y el movimiento lateral dentro de la red.
  • Las infecciones de ransomware a menudo se propagan a través de configuraciones débiles, credenciales comprometidas y protocolos heredados como SMBv1, aún habilitado en muchos entornos.

Acción Recomendada: Migrar urgentemente los sistemas EOL a versiones soportadas. Si la migración no es posible de inmediato, implementar segmentación de red estricta, deshabilitar protocolos obsoletos (ej. SMBv1), aplicar parches ESU (si aplica y está disponible) y asegurar copias de seguridad inmutables y aisladas.

Recomendaciones Operativas

Gestión de Parches y Actualizaciones

  • Asegurar que todos los sistemas operativos (Windows 10, 11, Ubuntu Server 24.04 LTS, RHEL 7/8, macOS 2015+) y software (FortiOS 7.2.12+, Proxmox VE 8.4.16+, Zabbix 7+, Zimbra Collaboration Professional 10+, ESET Antivirus) se mantengan con las últimas actualizaciones de seguridad y parches del proveedor.
  • Priorizar la aplicación de parches para vulnerabilidades "Explotadas activamente", como CVE-2026-32202 en sistemas Windows.
  • Establecer un plan de migración urgente para todos los sistemas EOL (Windows XP, 7, Server 2003/2008/2012, SQL Server Express 2005/2008). Si la migración no es inmediata, aplicar Extended Security Updates (ESU) donde estén disponibles y reforzar las medidas de seguridad perimetral y de red.

Seguridad de Identidad y Acceso

  • Implementar Autenticación Multifactor (MFA) en todas las cuentas de Office 365, acceso a FortiGate, Proxmox VE, Zimbra y Dell iDRAC.
  • Reforzar las políticas de contraseñas, exigiendo complejidad y rotación regular para todos los servicios, especialmente en Dell iDRAC y dispositivos Dahua.
  • Auditar y limitar el acceso administrativo con el principio de mínimo privilegio, especialmente en entornos de servidor y dispositivos de red.

Defensa contra Phishing y Seguridad de IoT

  • Realizar capacitaciones periódicas de concienciación sobre ciberseguridad para los usuarios de Office 365, enfocándose en la identificación de correos de phishing, suplantación de identidad y ataques PhaaS.
  • Configurar políticas robustas de seguridad de correo electrónico en FortiMail y Office 365 para detectar y bloquear campañas de suplantación.
  • Segmentar las redes para aislar dispositivos IoT (cámaras Dahua, FortiAP) del resto de la infraestructura crítica, y deshabilitar servicios no utilizados.
  • Cambiar las credenciales predeterminadas de fábrica en todos los dispositivos IoT.
